# 连接器总规范

## 目的
- 定义连接器的分类、生命周期、配置、安全、同步、原始落库和下游对接规则。
- 保证所有连接器都能以统一方式被配置、验证、执行、观测和扩展。

## 适用范围
- 适用于所有已存在和未来新增的连接器。
- 适用于前端连接器配置、`admin-api` 连接器管理、`sync-worker` 连接器执行、`raw` 数据落库和 dbt 对接任务。

## 规范正文

### 连接器分类
- 每个连接器在接入前必须先确定接入模式。
- v1.0 允许的接入模式必须限定为：
  - `scheduled pull`
    连接器按计划任务或手动时间窗拉取外部数据。
  - `manual parse/confirm`
    连接器先解析用户输入或上传材料，再由用户确认写入原始层。
- 新连接器不得在未定义接入模式的情况下进入实现阶段。

### 标识与目录规范
- 每个连接器必须拥有稳定、唯一的 `connector_id`。
- `connector_id` 必须使用 kebab-case。
- 同一 `connector_id` 必须在以下位置保持一致：
  - 前端配置目录与类型
  - `admin-api` 配置与接口路径
  - `sync-worker` 执行或内部接口
  - `raw` 数据中的 `connector_id`
  - dbt 下游映射
  - 专属文档名称与说明

### 生命周期规范
- 每个连接器都必须明确以下生命周期阶段：
  1. 定义连接器元数据
  2. 定义配置字段
  3. 实现配置保存与加密存储
  4. 实现连接验证
  5. 实现同步或解析流程
  6. 实现原始数据落库
  7. 实现下游 dbt 对接
  8. 实现测试与文档
- 任何跳过其中关键阶段的连接器都视为未完成接入。

### 配置规范
- 每个连接器必须明确最小配置字段集合，不得把非必要字段做成必填。
- 配置字段必须在前端和后端同时定义，且字段键名保持一致。
- 敏感字段必须以密文形式持久化。
- 保存配置前必须先通过连接验证，除非专属文档明确允许离线保存。
- 配置保存成功后，连接器状态必须进入可预测状态，不得出现“已保存但运行语义未知”。

### 验证规范
- 每个连接器都必须具备独立的验证能力。
- 验证必须覆盖：
  - 必填字段完整性
  - 凭证可用性
  - 外部依赖可达性
- 验证失败必须返回稳定错误码和清晰错误消息。
- 前端不得自行模拟“验证成功”作为保存前提，验证真相必须来自后端链路。

### 同步与任务规范
- 连接器必须先定义同步方式：
  - 自动调度
  - 手动触发
  - 手动解析后确认
- 同步任务必须具备显式状态流转：
  `queued -> running -> success/failed`
- 同一连接器实例的并发策略必须清晰，不得让多次运行互相覆盖真相。
- 任务记录必须保留窗口、计数、状态和错误信息等基础审计字段。

### 原始落库规范
- 任何连接器数据都必须先进入 `raw` 层，再进入 dbt 处理层。
- 原始落库必须保留适用的统一血缘字段，包括：
  - `account_id`
  - `connector_config_id`
  - `sync_task_id`
  - `connector_id`
  - `source_stream`
  - `external_id`
  - `source_record_date`
  - `source_record_at`
  - `source_updated_at`
  - `payload_hash`
  - `collected_at`
  - 原始载荷
- 连接器不得直接把消费层字段写成最终事实而绕开原始层。

### dbt 对接规范
- 每个连接器都必须定义：
  - 原始流清单
  - 对应的 staging 模型
  - 必要的 intermediate 模型
  - 是否需要 marts 模型
- 新连接器如果尚未进入 marts，也必须至少补齐 raw 到 staging 的规范路径。
- 任何新增原始流都必须同步规划下游模型与字段血缘。

### 安全与审计规范
- 连接器凭证不得出现在日志、前端持久化存储或错误返回中。
- 解析型连接器的上传内容必须有格式和类型约束。
- 涉及外部账号或第三方 API 的连接器必须具备失败可追踪性与最小暴露原则。

### 文档规范
- 每个正式连接器必须拥有一篇专属文档。
- 每个专属文档必须说明：
  - 接入模式
  - 配置字段
  - 验证规则
  - 同步规则
  - 原始流
  - 下游模型
  - 特殊限制
- 新增连接器必须同时补齐专属文档和模板派生内容。

## 边界与例外
- 解析型连接器可以不支持自动调度，但必须明确其手动确认和单次同步规则。
- 某些连接器可以没有 marts，但不得没有 raw 和 staging。
- 未被文档定义清楚的连接器不得直接进入生产实现。

## 实施清单
- 先确定接入模式。
- 再定义 `connector_id`、配置字段和验证规则。
- 再实现同步或解析任务与原始落库。
- 再补齐 dbt 对接、测试和专属文档。
- 提交前检查前后端、worker、raw、dbt、文档是否全部联动。
