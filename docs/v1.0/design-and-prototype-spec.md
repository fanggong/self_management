# 设计与原型规范

## 目的
- 定义 v1.0 产品界面的页面结构、交互模式和视觉约束。
- 保证新增页面、弹窗、列表和表单能够延续统一的信息架构和使用体验。

## 适用范围
- 适用于登录/注册、工作台、连接器、数据模型、日志、设置等正式页面与弹窗。
- 适用于新增页面、局部重构、组件设计、原型输出和 UI 验收。

## 规范正文

### 信息架构
- v1.0 页面体系必须分为两类：
  - 认证类页面
  - 已登录工作台页面
- 认证类页面必须只承载进入系统所需的最小流程，不得混入复杂业务配置。
- 工作台页面必须遵循“导航层 + 内容层 + 反馈层”的结构。
- 未被本规范正式定义的导航项视为预留，不得在任务中默认展开实现。

### 页面层级
- 一级层级必须保持稳定：
  - 认证入口
  - Dashboard
  - Connector
  - Data Model
  - System Settings
- 二级层级必须通过侧边导航、页签或明确的子模块入口呈现，不得隐藏在无标识的临时按钮后。
- 同一一级层级内的页面标题、面包屑和导航标签必须一致。

### 布局模板
- 系统必须统一使用两套布局模板：
  - `minimal`
    用于登录、注册等轻流程页面。
  - `authenticated`
    用于所有工作台页面。
- `minimal` 布局必须突出单卡片中心操作，不得引入复杂侧栏。
- `authenticated` 布局必须包含：
  - 侧边导航
  - 顶部栏
  - 面包屑
  - 主内容区域
  - 全局反馈层
- 主内容区域必须支持滚动，不得让页面整体在桌面端产生不可控双滚动。

### 导航规范
- 侧边导航必须按业务分组展示，不得把同层级能力杂糅为一组无语义列表。
- 当前页面必须在导航、标题和面包屑三处同时可感知。
- 父级导航项若包含子项，展开/收起行为必须明确，且同层只允许一个主要展开上下文。
- 导航项命名必须优先体现业务意图，而非技术实现细节。

### 工作台页面骨架
- 工作台页面必须优先采用以下骨架之一：
  - `概览页`
    指标卡片 + 图表/列表 + 详情面板
  - `配置页`
    列表概览 + 配置弹窗/表单
  - `任务页`
    过滤区 + 搜索区 + 任务列表/卡片 + 详情或操作入口
  - `数据模型页`
    过滤/搜索 + 表格 + 行级动作 + 详情/运行弹窗
- 同一页面内不得同时引入多个相互竞争的主交互中心。

### 表单规范
- 每个输入项必须具备：
  - 明确标签
  - 合理 placeholder
  - 必填与错误反馈
- 表单必须在字段级和提交级同时提供反馈：
  - 字段级用于即时校验
  - 提交级用于整体失败原因
- 必填项不得只靠 placeholder 表达。
- 敏感字段必须使用掩码输入，并关闭不必要的自动填充。
- 表单操作区必须固定提供主操作和取消操作，主操作按钮文案必须为动词。

### 弹窗规范
- 弹窗只应用于：
  - 配置编辑
  - 单次确认
  - 运行过程可视化
  - 不适合离开上下文的明细查看
- 弹窗必须具备：
  - 明确标题
  - 关闭条件
  - 忙碌态禁用规则
  - 成功/失败反馈
- 长流程弹窗必须把流程拆成清晰区块，不得把解析、编辑、确认混成无层次长表单。

### 表格与列表规范
- 列表页必须优先支持：
  - 搜索
  - 关键过滤
  - 状态可视化
  - 行级主要动作
- 表格列必须服务于决策，不得堆砌低价值字段。
- 状态字段必须使用统一的标签、颜色或图标语义。
- 复制、查看详情、运行、重试等操作必须与主列或操作列保持稳定位置。

### 状态反馈规范
- 页面必须显式定义以下状态：
  - `loading`
  - `empty`
  - `error`
  - `success`
  - `busy`
- Toast 只用于短暂反馈，不替代字段错误和区域错误。
- 可恢复错误应优先展示在出错区域内部，不得只抛全局提示。
- 长任务必须提供持续可见的进度或状态视图。

### 视觉与组件规范
- 视觉体系必须优先复用主题 token、语义色和组件变量，不得在业务页面中大量散落硬编码色值。
- 字体、圆角、阴影和间距必须保持统一节奏，不得按页面随意更换。
- 图标必须用于增强识别，不得替代文字。
- 品牌图、连接器图标和状态图标必须有统一尺寸与容器规则。

### 响应式规范
- 设计基准必须同时覆盖桌面端与移动端。
- 桌面端优先保留侧边导航和多栏结构。
- 移动端必须收敛为抽屉导航、单列内容和可滚动主区。
- 在小屏场景下，过滤器、表格和表单必须允许重排，不得出现不可达操作。

## 边界与例外
- 原型规范优先约束正式业务页面，不约束内部测试页和一次性调试面板。
- 连接器专属表单可以扩展字段，但不得破坏通用配置弹窗结构。
- 未定义为正式页面的预留模块不得通过补一段占位文案就视为完成设计。

## 实施清单
- 新增页面前先判断其属于哪种页面骨架。
- 新增弹窗前先确认是否必须留在当前上下文。
- 新增表单前先补齐标签、校验、错误和忙碌态。
- 新增列表前先定义搜索、过滤、空态和行级动作。
- 新增导航前先确认层级、标题、面包屑和激活态一致。
